Java AES加密的使用及简单原理

发表于 2015-05-11 | 分类于 technology |

通常客户端与服务器之间的通信数据是在网络上进行传输的，因此需要对它进行加密以保证安全。最近在一次实际开发中发现，服务器AES加密后的密文到iOS客户端后无法进行解密，于是深入google了一下AES加密。发现AES加密在跨平台跨语言的地方简直是个大坑！

AES(Advanced Encryption Standard)，高级加密标准，是美国政府用于替换DES的一种加密算法标准，Java SDK中包含了部分AES的实现，注意，是“部分”实现。其实几乎每种语言对AES都是“部分”实现，这也就导致了跨语言时产生的各种坑。

AES算法有3个等级：16字节的key对应AES-128，24字节的key对应AES-192，32字节的key对应AES-256。有6种加密模式：ECB,CBC,PCBC,CFB,OFB,CRT。最后还有无数种的填充模式：NoPadding，PKCS5Padding，PKCS7Padding，ISO10126Padding等。所以AES就产生了很多神奇的组合：

AES/CBC/NoPadding
AES/CBC/PKCS5Padding
AES/CBC/ISO10126Padding
AES/CFB/NoPadding
AES/CFB/PKCS5Padding
AES/CFB/ISO10126Padding
AES/ECB/NoPadding
AES/ECB/PKCS5Padding
AES/ECB/ISO10126Padding
...

其中常用的加密模式有下面这些：

  • ECB：是一种基础的加密方式，密文被分割成分组长度相等的块（不足补齐），然后单独一个个加密，一个个输出组成密文。
  • CBC：是一种循环模式，前一个分组的密文和当前分组的明文异或操作后再加密，这样做的目的是增强破解难度。
  • CFB/OFB实际上是一种反馈模式，目的也是增强破解的难度。

ECB和CBC的加密结果是不一样的，两者的模式不同，而且CBC会在第一个密码块运算时加入一个初始化向量IV。
